Claves primarias (Primary Key)


Una clave primaria (primary key) es un campo o conjunto de campos que identifica de forma única a cada fila de una tabla.

Caracteristicas de las claves primarias(Primary Key)


Unicidad: Cada fila de la tabla debe tener un valor único para la clave primaria.

Integridad: Solo puede existir una clave primaria en cada tabla, y el atributo sobre el que se define una clave primaria NO puede tener valores NULL.

Identificación: La clave primaria identifica de forma única cada fila de la tabla.

Indexación: La clave primaria crea un índice en la tabla, lo que permite realizar búsquedas rápidas y eficientes.

Claves primarias simples

Es una clave primaria que está compuesta por una sola columna.

CREATE TABLE usuarios (
id INT(7) NOT NULL PRIMARY KEY,
nombre VARCHAR(255),
correo VARCHAR(255)
);

Claves primarias compuestas


Se usan cuando no hay ningun atributo en la tabla que identifique cada registro de manera unica. Es posible que un conjunto de atributos pueda identificar cada registro de manera unica.

Claves primarias compuestas

En MySQL para establecer que una tabla tenga una clave compuesta, lo indicamos al final del create table:

CREATE TABLE vehiculos(
patente CHAR(6) NOT NULL,
tipo CHAR(4),
horallegada TIME NOT NULL,
horasalida TIME,
PRIMARY KEY(patente,horallegada)
);

Columanas Autoincrementadas

Tipo de Dato

En MySQL, una columna autoincrementada debe ser de tipo entero, como INT

Valor Predeterminado

Si al insertar una fila se omite el valor de una columna autoincrementada o se inserta un valor nulo para esa columna, MySQL calculará automáticamente el próximo valor asignando el valor más alto existente en esa columna y sumándole una unidad.

Uso como Clave Primaria

Estas columnas suelen utilizarse como claves primarias 'artificiales', lo que significa que no representan datos significativos del mundo real, sino que existen solo con el propósito de identificar de forma única cada fila en la tabla.
